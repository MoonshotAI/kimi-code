# agent-core 全面 i18n 改造方案

## 目标

为 `packages/agent-core` 引入轻量内置 i18n 系统，清除所有硬编码用户可见英文字符串，支持 en/zh 双语及运行时切换。

## 范围

| 类别 | 文件 | 处理方式 |
|------|------|----------|
| 错误标题 | `errors/codes.ts` 45 个 title | 提取为 i18n key |
| 死代码清理 | `errors/codes.ts` 58 个 action | 删除（无下游消费） |
| 插件校验 | `plugin/manifest.ts` ~22 message | 提取为 i18n key |
| 后台通知 | `agent/background/index.ts` 模板 | 提取为 i18n key |
| Shell 工具 | `tools/builtin/shell/bash.ts` 标题 | 提取为 i18n key |
| 工具超时 | `tools/builtin/file/grep.ts` 等 | 提取为 i18n key |
| Flag 描述 | `flags/registry.ts` title/desc | 提取为 i18n key |

## 实现步骤

### Step 1：创建 i18n 基础设施

**新建文件：** `packages/agent-core/src/i18n/`

```
i18n/
├── index.ts      # 公共 API：t(), setLocale(), getLocale()
├── runtime.ts    # 内部：locale 状态, resolveMessage(), interpolate()
├── types.ts      # TranslationKey, Locale, MessageValue 类型
├── en.ts         # 英文源（key 结构定义）
└── zh.ts         # 中文翻译
```

**`types.ts`**（类型定义）：
```ts
export type Locale = 'en' | 'zh';
type MessageValue = string | { [k: string]: MessageValue };
type Join<K, P> = K extends string | number
  ? P extends string | number ? `${K}.${P}` : never : never;
type Paths<T> = T extends MessageValue
  ? T extends string ? never
  : { [K in keyof T]-?: Join<K, Paths<T[K]>> | K }[keyof T] : never;
export type TranslationKey = Paths<typeof import('./en').default>;
```

**`runtime.ts`**（运行时）：
```ts
import { en } from './en';
import { zh } from './zh';
import type { Locale } from './types';

const messages: Record<Locale, object> = { en, zh };
let currentLocale: Locale = 'en';

export function setLocale(locale: Locale): void {
  currentLocale = locale;
}
export function getLocale(): Locale {
  return currentLocale;
}
function resolve(key: string, locale: Locale): string | undefined {
  // 按点径深入对象
}
```

**`index.ts`**（公共 API，仿 kimi-code 模式）：
```ts
export function t(key: TranslationKey | (string & {}), params?: Record<string, string | number>): string {
  let msg = resolve(key, currentLocale) ?? resolve(key, 'en') ?? key;
  return params ? msg.replace(/\{\{(\w+)\}\}/g, (_, n) => String(params[n] ?? `{{${n}}}`)) : msg;
}
export { setLocale, getLocale } from './runtime';
export type { Locale, TranslationKey } from './types';
```

**关键差异**：agent-core 不做 `process.env`（浏览器不安全），默认 `'en'`，由消费端通过 `setLocale()` 设置。

### Step 3：迁移 errors/codes.ts

**改造策略**：将硬编码 title 替换为 i18n key 引用。

```ts
// Before
'config.invalid': {
  title: 'Invalid configuration',
  action: 'Check config.toml and provider/model settings.',
  retryable: false,
  public: true,
}

// After
'config.invalid': {
  titleKey: 'errors.configInvalid.title',
  retryable: false,
  public: true,
}
```

**新增解析辅助函数**：
```ts
export function resolveErrorTitle(code: KimiErrorCode): string {
  const info = KIMI_ERROR_INFO[code];
  return info?.titleKey ? t(info.titleKey) : info?.fallbackTitle ?? code;
}
```

**删除 action 字段**（58 个死代码字符串）。

### Step 4：迁移 plugin/manifest.ts

将每个 `message:` 硬编码字符串替换为 `t()` 调用：

```ts
// Before
message: `No manifest at ${KIMI_PLUGIN_ROOT_PATH} or ${KIMI_PLUGIN_DIR_PATH}`

// After
message: t('plugin.manifestNotFound', {
  manifestPath: `${KIMI_PLUGIN_ROOT_PATH} or ${KIMI_PLUGIN_DIR_PATH}`
})
```

### Step 5：迁移 agent/background/index.ts

```ts
// Before
const baseLine = info.status === 'timed_out'
  ? `${info.description} timed out.
  : ...

// After
const baseLine = info.status === 'timed_out'
  ? t('background.timedOut', { description: info.description })
  : info.stopReason
    ? t('background.wasKilled', { description: info.description, reason: info.stopReason })
    : t('background.completed', { description: info.description });
```

### Step 6：迁移 tools/builtin/shell/bash.ts

```ts
// Before
title: 'Background task started'

// After
title: t('shell.backgroundStarted')
```

### Step 7：迁移工具超时字符串

```ts
// grep.ts Before
`Grep timed out after ${String(DEFAULT_TIMEOUT_MS / 1000)}s. Try a more specific path or pattern.`

// After
t('tools.grepTimedOut', { seconds: String(DEFAULT_TIMEOUT_MS / 1000) })
```

### Step 8：迁移 flags/registry.ts

将 title/description 改为 key 引用，渲染时解析：

```ts
// Before
{ id: 'tool-select', title: 'Tool select...', description: '...' }

// After
{ id: 'tool-select', titleKey: 'flags.toolSelectTitle', descKey: 'flags.toolSelectDesc' }
```

### Step 9：消费端集成

**TUI (`apps/kimi-code`)**：
- 启动时调用 `import { setLocale } from '@moonshot-ai/agent-core/i18n'` 并传入当前 locale
- `startup-error.ts` 改为使用 `resolveErrorTitle(error.code)` 而非 `info.title`
- Flag 显示组件改为 `t(flagDef.titleKey)` — 复用 TUI 自己的 i18n 系统（需导入 agent-core 的 en/zh 作为 key 源）

**Web (`apps/kimi-web`)**：
- 可选：将 agent-core 的 en/zh 注册为 vue-i18n 消息源
- 或保持现有 `AGENT_ERROR_TITLE_KEYS` 映射，逐步迁移

## 验证方式

1. **TypeScript 编译**：`pnpm --filter @moonshot-ai/agent-core run build` 无错误
2. **单元测试**：`pnpm --filter @moonshot-ai/agent-core test` 全部通过
3. **类型检查**：TranslationKey 提供自动补全，无效 key 编译报错
4. **运行时验证**：
   - TUI 启动时 `setLocale('zh')`，触发一个配置错误，确认标题显示中文
   - 运行一个超时 grep，确认超时提示为中文
   - 加载一个无效插件，确认校验消息为中文
5. **回归测试**：`pnpm --filter @moonshot-ai/klient test` 确保 SDK 消费端不受影响
