04

有意保留的偏离

这些是与 spec 不一致、但经确认有意保留的项。

TodoCard 头部栏

功能增强

spec 的 .p-todo 是纯行列表；当前 TodoCard 顶部多了一栏（图标 +「To-do」标题 + done/total 计数 + 折叠箭头）。属于功能增强，样式已统一，保留。

气泡圆角 / 字号

token 化
项spec当前原因
用户气泡圆角18/18/5/1816/16/6/16用刻度内 token（18 / 5 不在 radius 刻度内）
气泡字号15px16px用 content-font-size
正文字号15px16px同上

ThinkingBlock

结构不同

spec 的 .p-thinking 是一行流式指示器（对应当前的 ActivityNotice）。当前 ThinkingBlock 是可折叠的滚动窗口（在右侧栏展开全文），属于增强，保留。

Token 层偏离

规范差异
tokenspec当前影响
--weight-bold700600spec 的 700 标题用了字面量 700

subagent thinking / 工具调用

不改 projector

subagent 的 thinking delta、工具调用细节当前被 projector 丢弃，只保留合成的 Calling xxx 进度行 + 最终 summary。按你的要求，projector 不改——参数与结果对 replay 足够，thinking 等额外信息不收录。

check:style 误报

检测脚本

GoalStrip.vue:165 的 border-radius: inherit、SideChatPanel.vue 的两处 var(--r-sm, 8px) 被 radius-from-scale 误报。是检测脚本未白名单 inherit / var() fallback，非代码问题。

待处理

尚未处理、需要时再说的项。

工具组 summary 的耗时

小

spec 的工具组 summary 行显示「· 已完成 · 0.8s」。当前 ToolGroup 只显示「· 已完成」——ToolCall 只有格式化后的 timing 字符串（如 12ms / 0.2s），没有原始毫秒，聚合总耗时需解析字符串，暂未做。

check:style 误报清零

低优先级

可改 scripts/check-style.mjs 的 radius-from-scale 规则：白名单 inherit、并把 var(--x, Npx) 的 fallback 值纳入判定。或改代码避开。优先级低（不影响功能、不 fail）。
