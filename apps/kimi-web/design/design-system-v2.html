Design System v2 · 提案

统一卡片语言

把工具卡片那套「扁平、紧凑、克制」的语言立为全站标准：所有卡片统一为扁平壳（无阴影）+ radius-md； 头部按语意区分（操作卡 = 等宽紧凑、关注卡 = 彩色色带）；分组统一为「组拥有边框、行用发际线分隔」。

📐
这是路线 3 的可视化提案：统一壳（消除「同一类东西两种壳」），保留必要的头部区分。所有示例都是真实渲染（非截图），可直接对照。
01

统一卡片壳

全站所有卡片共用同一个壳：扁平（无阴影）+ surface 底 + 1px line 描边 + radius-md (8px)。不再有的卡片有阴影、有的没有。

统一壳 · v2-card
◇ 卡片标题 meta
卡片正文。所有卡片都用这个壳：扁平、8px 圆角、1px 描边、无阴影。

规则

  • 无阴影：卡片不靠阴影堆层级，靠 1px 描边 + 底色。
  • radius-md (8px)：所有卡片统一 8px 圆角（不再有的 8、有的 12）。
  • surface 底：默认 --color-surface（不是 raised）。
  • 头部按语意区分：见 02 / 03。
⚠
这和当前 design system v1（卡片 shadow-sm + radius-lg）不同。v2 把工具卡片的「扁平 + radius-md」立为标准。
02

操作卡（工具卡片）

工具调用、Agent 组、Todo、Swarm 这类「操作/内联」卡片：等宽紧凑头 + 状态点 + 可展开的 sunken 详情。

操作卡 · 单条
⌘ bash pnpm --filter web typecheck 1.2s ›
✓ typecheck passed in 1.2s no errors found

规则

  • 头部 30px、等宽：icon + 名称 + 参数（省略）+ 状态点 + 时间 + 展开箭头。
  • 状态点：绿 = 完成、红 = 失败、accent 脉冲 = 运行中（见 05）。
  • 详情 sunken：展开后是 surface-sunken 浅底、缩进、等宽。
  • 无彩色色带：操作卡的头部是透明的（hover 才变浅），不像关注卡有色带。
03

关注卡（决策卡）

Question / Approval 这类「需要用户关注/决策」的卡片：壳仍是统一的扁平 + radius-md，但头部用彩色色带 + 非等宽标题来提示「这需要你处理」。这是有意的区分。

关注卡 · Question（accent）
? 需要确认 步骤 1/2
要把登录接口改成 JWT 吗？这会改动 auth.ts 和 middleware.ts。
关注卡 · Approval（warning）
! 需要审批 required
即将执行 rm -rf node_modules，是否允许？

规则

  • 壳统一：和操作卡一样的扁平 + radius-md（不再有阴影）。
  • 色带头部：accent（question）/ warning（approval）软色头带 + 同色标题。
  • 非等宽标题：用 font-ui + semibold，和操作卡的等宽紧凑头形成「有意区分」。
  • 意图：色带 =「这需要你处理」，操作卡无头带 =「这是后台操作」。
04

分组（堆叠）

多个相关项归组时，组拥有外边框 + 圆角，组内每行扁平、只用 1px 发际线分隔。这是你喜欢的「多工具调用集合」的做法，应推广到所有分组。

分组 · 组拥有边框、行用发际线分隔
☰ 3 个工具调用 · 运行中 ›
⌘ bash pnpm typecheck
✎ edit auth.ts
⌘ bash pnpm test

规则

  • 组拥有边框 + 圆角：组是一个整体（一个壳）。
  • 行扁平：组内每行无自己的边框/圆角/阴影。
  • 发际线分隔：行与行之间只用 1px line 分隔。
  • 组头可折叠：组头 hover 变浅、箭头旋转 90°。
05

状态指示器

状态用统一的「颜色词汇」：绿 = 成功/完成、红 = 失败、accent = 进行中/激活、muted = 闲置。形态可按场景变（点、图标、复选框），但颜色词汇不变。

状态点
成功 / 完成 失败 进行中 闲置
  • 颜色词汇统一：success=绿、danger=红、accent=蓝、muted=灰。
  • 形态可按场景：工具用点/图标、Todo 用复选框、Question 用步骤点——但颜色含义一致。

落地清单

如果认可这个方向，按这个顺序改。

  • 1. 统一壳：Card.vue 去 shadow-sm、radius-lg → radius-md。
  • 2. 统一壳：TodoCard / SwarmCard 去 shadow-sm、radius-lg → radius-md。
  • 3. 保留头部区分：Question/Approval 的色带头部不动（关注卡的有意区分）。
  • 4. 同步 design system v1：把卡片的 shadow-sm 去掉、radius-lg 改 radius-md，与 v2 对齐。
  • 5. 分组：AgentGroup 的组内从「网格分卡」考虑是否也改成「堆叠发际线」（可选，影响较大）。
