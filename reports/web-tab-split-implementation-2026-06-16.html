结论：工作区现在稳定在 对话 / 文件 两个持久主 Tab。Preview、BTW 只在被使用时作为分屏里的临时 Tab 出现；Thinking、Compaction、Subagent 详情、移动端文件/媒体预览统一进右侧详情层。 Tasks / Todo / Terminal 没有被重新加回一级主 Tab。同时补齐了 tab/分屏的可访问性、键盘路径、按钮热区、弹层焦点和 diff 配色。共 6 个提交，全部测试 + 类型检查通过。

1最终视图分层模型

这是收敛后的目标模型，三层各司其职，临时层永远不污染持久主 Tab：

持久主 Tab 对话 / 文件
每个分屏组（ViewGroup）固定只有 chat 与 files 两个 Tab。 数据来源 usePaneLayout.ts 的 ALL_VIEWS = ['chat','files']； normalizeLayout() 会把任何持久化里残留的 tasks/todo/terminal 过滤掉，所以刷新后也回不到旧的一级 Tab。
临时 Tab（分屏内） Preview / BTW
只在“正在用”时出现：openPreview() / openBtw() 会把当前组分屏出一个专用临时组，并在 TabBar 上通过 hasPreview / hasBtw 显示对应 Tab；关闭即折叠回去，且不写入持久化。它们不会抢占底部输入框或移动端的主 Tab 状态。
右侧详情层 Thinking / Compaction / Subagent / 移动端预览
App 的 aside.global-preview，一次只显示一个：Thinking 文本、Compaction 摘要、Subagent 详情，移动端额外承载文件/媒体预览。现在是带标签的 role="complementary" 区域，Escape 关闭，关闭按钮统一带 aria-label。
