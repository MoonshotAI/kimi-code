设计要点

· 顶层每一项是一个 worktree（分支名 + 所属 repo），不再是 workspace。

· 排序：有运行中 / 待处理 session 的 worktree 归到「活跃」区置顶；其余按最近 session 时间倒序。

· 每个 worktree 头部显示 分支 + repo + PR，颜色圆点来自分支（等明度）；有运行 session 时多一个绿色 live 点。

· main checkout 也作为一个 worktree 项（分支 main），但默认排在活跃 worktree 之后。

· 这是「合二为一」：workspace 退化为 worktree 头上的 repo 小字，侧边栏只剩 worktree 这一层。

· 非 git / 还没识别出分支的 session 可放在末尾一个「未归类」组。
